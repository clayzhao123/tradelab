# tradelab 前端重设计 Prompt

> 将本文件完整内容粘贴给 Cursor / Claude 使用。

---

## 项目背景

你正在重设计一个名为 **tradelab** 的 Web 端 Crypto 模拟交易台（Paper Trading）。  
技术栈：**Vanilla JS + HTML5 Canvas**，单页应用（SPA），连接 Node.js 后端（REST + WebSocket）。  
目标：在不改动后端逻辑和数据结构的前提下，将前端 `app.js` + `index.html` + `styles.css` 重构为一个具备高可读性、高易用性的专业 Trading Dashboard。

## 设计哲学：Claude.ai 风格

参照 Claude.ai 的设计语言，核心原则如下：

### 1. 整体基调
- **克制的精致感**：不张扬，不花哨，细节处理精到
- **信息优先**：每一个元素都应服务于数据可读性，装饰元素极度克制
- **温暖中性**：不用冷灰，用带有微微暖色调的中性色（如 `#F5F4F0`、`#2C2C2A`）
- **全局双模式**：Light / Dark 双模式，CSS 变量驱动，一套变量完整覆盖两种模式

### 2. 色彩系统（CSS 变量）

在 `:root` 和 `[data-theme="dark"]` 中定义以下变量：


### 3. 字体规范
- **数字**：一律使用 `var(--font-mono)`，字重 500，便于对齐和扫描
- **正文**：`var(--font-sans)`，字重 400
- **标签/标题**：字重 500，letter-spacing: 0.01em
- **禁止**使用 `font-weight: 700` 以上
- **价格大数字**：font-size: 20-24px，mono，保留 2 位小数
- **百分比**：font-size: 12-13px，带颜色 `--color-up/down`

## 布局结构

采用三列布局，全屏固定高度，不出现页面滚动：

```
┌─────────────────────────────────────────────────────┐
│  TopBar：Portfolio 总览 + 风险状态 + 控制开关        │  48px
├──────────┬────────────────────────┬──────────────────┤
│          │                        │                  │
│  Left    │   Main（图表区）       │  Right Panel     │
│  Sidebar │                        │  （订单/账户）   │
│  240px   │   flex: 1              │  300px           │
│          │                        │                  │
│  币种列表│   K线图 Canvas         │  活跃订单        │
│  + 价格  │   + 扫描结果列表       │  + 成交记录      │
│  + 评分  │                        │  + 风险事件      │
│          │                        │                  │
└──────────┴────────────────────────┴──────────────────┘
```

- **TopBar**：高度 48px，sticky，背景 `--bg-surface`，底部 1px `--border-subtle`
- **Left Sidebar**：宽度 240px，背景 `--bg-surface`，右侧 1px `--border-subtle`
- **Main Area**：`flex: 1`，背景 `--bg-page`，内部 padding 12px
- **Right Panel**：宽度 300px，背景 `--bg-surface`，左侧 1px `--border-subtle`

## K线图规范（Canvas）

使用 HTML5 Canvas 绘制，适配 `--bg-page` 背景：

```javascript
// 颜色从 CSS 变量读取（通过 getComputedStyle）
const style = getComputedStyle(document.documentElement);
const colorUp   = style.getPropertyValue('--color-up').trim();
const colorDown = style.getPropertyValue('--color-down').trim();
const colorGrid = style.getPropertyValue('--border-subtle').trim();
const colorText = style.getPropertyValue('--text-tertiary').trim();

// 蜡烛样式
// 实体：涨 colorUp / 跌 colorDown，圆角 1px
// 影线：同色，透明度 0.6
// 网格线：colorGrid，1px，dashed (4,4)
// 坐标轴文字：colorText，11px，var(--font-mono)
```

时间框架切换 Tab：`1m | 5m | 15m | 1H`，使用 pill 样式切换器。

## 动效规范

- **所有状态切换**：`transition: background 120ms ease, color 120ms ease`
- **价格数字更新**：数字变化时，播放 200ms fade（opacity 0.5 → 1）+ 颜色闪烁（绿/红 → 正常，300ms）
- **运行状态绿点**：`@keyframes pulse` 循环，radius 放大 + 透明度下降
- **新订单/成交**：行从 `--color-accent-bg` 渐变回 transparent，600ms
- **禁止**：禁止使用任何 entrance animation 和大幅位移动效

```css
@keyframes pulse-dot {
  0%   { box-shadow: 0 0 0 0 rgba(26,122,82,0.5); }
  70%  { box-shadow: 0 0 0 6px rgba(26,122,82,0); }
  100% { box-shadow: 0 0 0 0 rgba(26,122,82,0); }
}
@keyframes price-flash-up {
  0%   { color: var(--color-up); }
  100% { color: var(--text-primary); }
}
@keyframes price-flash-down {
  0%   { color: var(--color-down); }
  100% { color: var(--text-primary); }
}
```

## 排版 & 对齐原则

### 数字对齐
所有价格、数量、PnL 数字必须**右对齐**，使用 `text-align: right`。  
同列数字使用相同小数位数，不足时补 `0`（如 `43,280.50` 而非 `43280.5`）。

### 信息密度
- 左 Sidebar：中密度，每行 52px，清晰但紧凑
- 右 Panel：中低密度，每行 64px，状态一目了然
- 禁止出现横向滚动条

### 分割线使用原则
- **优先使用间距**（gap/margin）分隔内容，而非添加分割线
- 分割线只在跨越不同语义区块时使用，颜色 `--border-subtle`，高度 1px

## 需要重点改进的具体问题

以下是当前前端的**已知痛点**，请在重构时逐一修复：

1. **价格数字可读性差**：统一使用 mono 字体，固定小数位，加千分位分隔符
2. **状态反馈不明确**：Running/Paused 状态需要明显的视觉指示器
3. **信息层级模糊**：通过字体大小、颜色、间距建立清晰的 primary → secondary → tertiary 层级
4. **扫描结果难以扫描**：改为结构化表格，评分加颜色 badge，方向加图标
5. **订单状态不直观**：OPEN/PARTIAL/FILLED/CANCELLED 需要不同颜色的 badge
6. **图表与行情数据脱节**：点击 Sidebar 中的币种，图表应立即切换
7. **风险事件被淹没**：风险事件应在显眼位置以 banner 形式展示
8. **深色模式缺失或不完整**：实现完整的 CSS 变量驱动 dark mode
